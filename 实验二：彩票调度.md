###1.pstat.h

最先是pstat.h文件。这部分非常简单，直接复制实验说明里的代码即可。
```c
struct pstat {
int inuse[NPROC]; // 是否可运行（1或0）
int tickets[NPROC]; // 此进程拥有的彩票数
int pid[NPROC]; // 每个进程的进程ID
int ticks[NPROC]; // 每个进程已经运行的CPU时间片数量（被选择运行的次数）
};
//NPROC:最大进程数
```

### 2.1实现testTick.c

按照实验说明，该程序主要是接受命令并把数字设为tickets的值。

当时就联想到了实验一的内容，因此仿照实验一的代码写就可以了：
```c
#include "types.h"
#include "stat.h"
#include "user.h"

int main(int argc, char *argv[]) {
    
    //输入错误时提示正确语法
    if (argc < 2) {
        printf(1, "Usage: testTicket <tickets>\n");
        exit();
    }
    
    //获取票数
    int tickets = atoi(argv[1]);
    
    if (settickets(tickets) < 0) {
        printf(1, "settickets failed!\n");
    }
    else {
        printf(1, "settickets(%d) succeeded\n", tickets);
    }
    
    while(1);  // 无限循环
    exit();
}
```
注意到此处有while(1)的无线循环，它的目的是让程序一直运行下去，以此来去“抢彩票”。

### 2.2实现testProcInfo.c

同样由实验说明可以得知，其主要作用是显示
进程ID、是否可运行、分配的彩票数和已累积的CPU时间片数（ticks）

也很容易写出：
```c
#include "types.h"
#include "stat.h"
#include "user.h"
#include "pstat.h"

int main(int argc, char *argv[]) {
    struct pstat ps;

    //获取进程信息
    if (getpinfo(&ps) < 0) {
        printf(1, "getpinfo failed\n");
        exit();
    }

    printf(1, "\nProcessID | Runnable | Tickets | Ticks\n");
  

    for (int i = 0; i < 64; i++) { 
        if (ps.inuse[i]) {
            printf(1, "%d         | %d        | %d       | %d\n", 
                    ps.pid[i], ps.inuse[i], ps.tickets[i], ps.ticks[i]);
        }
    }

    exit();
}
```

### 3.引入伪随机数生成器

我先是了解了一下一些伪随机数的生成逻辑。

我使用的LCG来生成，其本质就是先定义一个f(x)=（a*x+b）mod c，再选取一个基数x0。

接下来不断用递推式x(i+1)=f(xi)来生成新的数就可以了。

很惭愧地说，这是我第一次写.h文件，所以我确实花了一些时间来了解这部分内容。

最终代码如下：
```c
#ifndef _RAND_H_
#define _RAND_H_

void rand_init(unsigned int seed);
unsigned int rand(void);
unsigned int random_at_most(unsigned int max);

#endif
```
其实本质上就是先申明了rand.c文件里应该要包含哪些东西，或者说.h文件是用来提供接口的，而.c文件就是具体实现的方式。

代码中首尾部分是用于保护的，避免被重复定义。

rand.c文件的内容如下：
```c
#include "rand.h"
#include "types.h"

static uint next=1;

//初始化
void rand_init(unsigned seed){
    next=seed;
}

//生成伪随机数
uint rand(void){
    next=next*142857+769230;
    return (unsigned)(next/58823)%5294117;
}

//限制在给定大小内
uint random_at_most(uint max){
    return (rand()%(max+1));
}
```
这里的参数看上去很奇怪，实际上我是用了某些分数的循环节，比如:

1/7=0.142857......

1/13=0.076923......

1/17=0.58823 5294117 (647)......

###4.实现settickets的系统调用

根据实验说明修改proc.h中的struct proc如下：

```
// Per-process state
struct proc {
  uint sz;                     // Size of process memory (bytes)
  pde_t* pgdir;                // Page table
  char *kstack;                // Bottom of kernel stack for this process
  enum procstate state;        // Process state
  int pid;                     // Process ID
  struct proc *parent;         // Parent process
  struct trapframe *tf;        // Trap frame for current syscall
  struct context *context;     // swtch() here to run process
  void *chan;                  // If non-zero, sleeping on chan
  int killed;                  // If non-zero, have been killed
  struct file *ofile[NOFILE];  // Open files
  struct inode *cwd;           // Current directory
  char name[16];               // Process name (debugging)
  int tickets;                 // 跟踪每个进程持有的彩票数量 
  int tick;                    // 当前运行的CPU时间片
};
```c

接着在proc.h中添加申明：

```
int settickets(int);
```c

显然，这个函数的作用是给进程设置票数，但是我要从哪里知道我现在在哪个进程呢？

幸好，在proc.c文件中有一个myproc()函数：
```c
myproc(void) {
  struct cpu *c;
  struct proc *p;
  pushcli();
  c = mycpu();
  p = c->proc;
  popcli();
  return p;
}
```
使用myproc可以获得当前进程的指针，于是可以顺利更改票数。

然后在proc.c文件中写出其完整代码：
```c
int settickets(int n){
  //不能少于1
  if (n < 1){ 
    return -1;
  }
  //设置当前进程的彩票数
  myproc()->tickets = n;
  return 0;
}
```

然后是在sysproc.c中编写sys_settickets()函数。

实验说明提到可以调用argint()函数，我在syscall.c中找到了定义：
```c
int
argint(int n, int *ip)
{
  return fetchint((myproc()->tf->esp) + 4 + 4*n, ip);
}
```
这个函数的作用就是让内核能够安全地读取用户程序传递给系统调用的参数。
